# Contributing to Experiment Atlas

Thanks for helping make the experiment workflow more useful.

## Before opening a pull request

1. Keep the change focused on one user or developer outcome.
2. Run `npm test`.
3. Update the README when behavior or setup changes.
4. Use a commit message that explains the change, for example:
   - `feat: add experiment tags`
   - `fix: reject invalid metrics`
   - `docs: clarify local setup`

## Good next contributions

- replace the JSON store with SQLite
- add experiment tags and comparison views
- connect a PyTorch training script to the API
- add pagination once the dataset grows
- add a small authentication layer for shared workspaces
